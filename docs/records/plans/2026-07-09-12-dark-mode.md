# テンプレートのダークモード（藍墨）実装プラン（Issue #12）

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 和モダン・エディトリアルのテンプレートに、OS自動追従（`prefers-color-scheme: dark`）のダークモード「藍墨」を追加する。ライトの見た目は維持し、全コントラストWCAG AAを満たす。

**Architecture:** CSSのみで実現（JS不使用のためFOUCなし）。`global.css` に背景/面/境界トークンを新設して既存ハードコード色を集約し、`@media (prefers-color-scheme: dark)` で全色トークンと季節アクセント4セレクタを藍墨値に差し替える。季節ヒーローは `SeasonalHero.astro` の `<style>` にダーク版グラデを追加する。

**Tech Stack:** Astro 6（静的サイト）、素のCSS。追加依存なし。

**設計spec:** `docs/records/specs/2026-07-09-12-dark-mode-design.md`

### Task 2: `@media dark` で全トークンと季節アクセントを藍墨化

**Files:**
- Modify: `src/styles/global.css`（季節アクセント定義ブロックの直後。`html[data-season="winter"]` ブロックの後ろ）

- [ ] **Step 1: ダーク上書きブロックを追加**

`src/styles/global.css` の `html[data-season="winter"] { --season-accent: #87a3bc; }` ブロックの直後に、以下を追加する。**季節アクセントは必ず `:root`（春）＋ `html[data-season]`×3 の同一セレクタで書く**（`@media` は詳細度を上げないため、`:root` だけだと `html[data-season="summer"]` に負けて夏で切り替わらない）。

```css
/* ===== ダークモード（藍墨）: prefers-color-scheme 自動追従 ===== */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #1b1e24;
    --color-surface: #232730;
    --color-ink: #e8e6e1; /* 生成りのオフホワイト（対地13.4:1） */
    --color-ink-soft: #c3c1bb; /* 対地9.3:1 */
    --color-muted: #9a9891; /* 対地5.8:1 */
    --color-border: #3a3f48;
    --season-accent: #e79aae; /* 春 */
  }
  html[data-season="summer"] {
    --season-accent: #5cbdca;
  }
  html[data-season="autumn"] {
    --season-accent: #e0904f;
  }
  html[data-season="winter"] {
    --season-accent: #a3bdd4;
  }
}
```

- [ ] **Step 2: ビルド確認**

Run: `npm run build`
Expected: エラー・警告なく成功。

- [ ] **Step 3: ダーク切替の目視確認**

Run: `npm run dev`。Chrome DevTools の Rendering パネルで「Emulate CSS media prefers-color-scheme: dark」を有効にしてリロード。
Expected: 背景が藍墨（#1b1e24）、文字が生成り白、Skills/Contact・見出しの季節アクセントがダーク版になる。ライトに戻すと元通り。

- [ ] **Step 4: コミット**
